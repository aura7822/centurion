00 Prerequisites

Before anything, confirm you have these on your Fedora machine:

  • ✓ Fedora 38 / 39 / 40 (or newer)
  • ✓ At least 4GB RAM, 10GB free disk
  • ✓ Internet connection (for downloading models and deps)
  • ✓ A webcam (for live testing)
  • ✓ sudo access

01 Install Dependencies

Run the setup script — it handles everything with dnf:

$ cd centurion/
$ chmod +x scripts/setup_env.sh
$ sudo ./scripts/setup_env.sh

If you prefer manual installation, here are the individual packages:

# Core build tools
$ sudo dnf install -y gcc-c++ cmake make git wget curl

# OpenCV with DNN support
$ sudo dnf install -y opencv-devel python3-opencv

# OpenSSL + CURL
$ sudo dnf install -y openssl-devel libcurl-devel

# MongoDB server
$ sudo dnf install -y mongodb-org

# MongoDB C++ driver
$ sudo dnf install -y mongo-cxx-driver-devel

# Tor
$ sudo dnf install -y tor

# Node.js (for blockchain)
$ sudo dnf install -y nodejs npm

Download required single-header libraries

# cpp-httplib (HTTP server, no Boost needed)
$ wget -O backend_cpp/include/httplib.h \
  "https://raw.githubusercontent.com/yhirose/cpp-httplib/master/httplib.h"

# nlohmann/json (JSON parsing)
$ wget -O backend_cpp/include/json.hpp \
  "https://github.com/nlohmann/json/releases/download/v3.11.3/json.hpp"
NOTE — If mongo-cxx-driver-devel is not in your Fedora repos, the setup script automatically builds it from source. This takes about 5 minutes.

02 Download AI Models

Run the model download helper first — it grabs everything that's freely available:

$ chmod +x scripts/download_models.sh
$ ./scripts/download_models.sh

Then manually download the face recognition model (required):

FileWhere to get itStatus
facenet_embedding.onnx Download ArcFace from onnx/models → rename to this REQUIRED
age_gender_model.onnx Script downloads Caffe version. For ONNX: yu4u/age-gender-estimation REQUIRED
ethnicity_model.onnx dchen236/FairFace — convert to ONNX OPTIONAL
anti_spoofing_model.onnx Silent-Face-Anti-Spoofing OPTIONAL
face_detection_yunet_2023mar.onnx Script downloads automatically AUTO

All models go into: backend_cpp/models/

Quickest face recognition model (ArcFace)

# Download ArcFace R100 ONNX (MIT license)
$ pip3 install insightface onnxruntime
$ python3 -c "
  import insightface
  m = insightface.app.FaceAnalysis(name='buffalo_l')
  m.prepare(ctx_id=-1)
  print('Models downloaded to ~/.insightface/models/')
"

# Then copy the w600k_r50.onnx as your face embedding model
$ cp ~/.insightface/models/buffalo_l/w600k_r50.onnx \
    backend_cpp/models/facenet_embedding.onnx

03 Build the C++ Backend

$ cd backend_cpp/
$ mkdir -p build && cd build
$ cmake -DCMAKE_BUILD_TYPE=Release ..
$ make -j$(nproc)
$ cd ../..
SUCCESS looks like:
[100%] Linking CXX executable centurion
[100%] Built target centurion

Verify the binary

$ ls -lh backend_cpp/build/centurion
# Should see something like: -rwxr-xr-x ... 8.2M centurion

04 Initialize MongoDB

Start MongoDB

$ sudo systemctl start mongod
$ sudo systemctl enable mongod

# Verify it's running
$ sudo systemctl status mongod
# You should see: Active: active (running)

Run the init script

$ mongosh --file database/init_mongo.js
Expected output:
✓ users collection ready
✓ face_embeddings collection ready
✓ access_logs collection ready
✓ Centurion® MongoDB init complete!

Verify in mongosh

$ mongosh
> use centurion
> show collections
# Should list: access_logs, face_embeddings, users, threat_scores, ...
> exit

05 Configure .env

$ cp .env.example .env
$ nano .env

The minimum you need to change:

# MongoDB — defaults work if running locally
MONGO_URI=mongodb://localhost:27017
MONGO_DB=centurion

# Port
CENTURION_PORT=8080

# Model and log directories
MODEL_DIR=models/
LOG_DIR=logs/
SNAPSHOT_DIR=snapshots/

# GPIO — set true only on Raspberry Pi
GPIO_ENABLED=false
IMPORTANT — If you plan to use blockchain logging, also fill in PRIVATE_KEY and RPC_URL. Otherwise leave blank — the backend works without blockchain.

06 Start the Backend

Option A — Direct (see live output)

$ cd backend_cpp/
# Load env vars then run
$ export $(grep -v '^#' ../.env | xargs)
$ ./build/centurion

Option B — systemd service (background, auto-restart)

$ sudo systemctl start centurion

# Watch live logs
$ sudo journalctl -u centurion -f

# Stop
$ sudo systemctl stop centurion

T2 Enroll a Face

Capture a photo of a person and enroll them as an authorised user. Use your webcam or any JPEG photo.

Step 1 — Convert your photo to base64

# Take a photo with your webcam (requires fswebcam)
$ sudo dnf install -y fswebcam
$ fswebcam -r 640x480 --no-banner test_face.jpg

# OR use an existing photo — convert to base64
$ base64 -w 0 test_face.jpg > face_b64.txt

Step 2 — Send enroll request

$ curl -s -X POST http://localhost:8080/api/enroll \
  -H "Content-Type: application/json" \
  -d "{\"userId\": \"alice\", \"image\": \"$(cat face_b64.txt)\"}" \
  | python3 -m json.tool
Expected response:
{"success": true, "userId": "alice"}

Verify in MongoDB

$ mongosh centurion --eval "db.face_embeddings.find({},{userId:1,enrolledAt:1})"

T3 Identify a Face

This is the main AI pipeline — face detection → embedding → matching → demographics → liveness → IoT trigger.

# Take a new photo and identify it
$ fswebcam -r 640x480 --no-banner identify_test.jpg
$ base64 -w 0 identify_test.jpg > id_b64.txt

$ curl -s -X POST http://localhost:8080/api/identify \
  -H "Content-Type: application/json" \
  -d "{\"image\": \"$(cat id_b64.txt)\"}" \
  | python3 -m json.tool
Expected response (authorised):
{
  "eventId": "evt-1234567890",
  "authorized": true,
  "userId": "alice",
  "confidence": 0.9234,
  "liveness": { "isLive": true, "score": 0.87 },
  "demographics": { "age": 28, "gender": "Female", "ethnicity": "..." },
  "iot": { "greenLED": true, "doorUnlocked": true, ... }
}

Test with an unknown face (should be rejected)

# Download a random portrait photo and test it
$ wget -q -O unknown.jpg "https://thispersondoesnotexist.com"
$ base64 -w 0 unknown.jpg > unknown_b64.txt
$ curl -s -X POST http://localhost:8080/api/identify \

!! Common Errors & Fixes

MongoDB connection failed

[MongoDB] Connection failed: No connection could be made
$ sudo systemctl start mongod
$ sudo systemctl status mongod
# If SELinux is blocking it:
$ sudo setsebool -P mongod_can_network_connect 1

Model not found / Build fails

[ERROR] No face embedding model found in models/
# You must place your ONNX model before running
$ ls backend_cpp/models/
# Must contain at minimum: facenet_embedding.onnx

cmake: mongocxx not found

CMake Error: MongoDB C++ driver not found!
# Try installing from dnf first
$ sudo dnf install -y mongo-cxx-driver-devel

# If not available, build from source:
$ sudo dnf install -y mongo-c-driver-devel cmake openssl-devel
$ wget https://github.com/mongodb/mongo-cxx-driver/releases/download/r3.10.1/mongo-cxx-driver-r3.10.1.tar.gz
$ tar xzf mongo-cxx-driver-r3.10.1.tar.gz
$ cd mongo-cxx-driver-r3.10.1/build
$ cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr/local ..
$ make -j$(nproc) && sudo make install && sudo ldconfig

Port 8080 blocked (SELinux / firewalld)

$ sudo semanage port -a -t http_port_t -p tcp 8080
$ sudo firewall-cmd --permanent --add-port=8080/tcp
$ sudo firewall-cmd --reload

httplib.h missing

$ wget -O backend_cpp/include/httplib.h \
  "https://raw.githubusercontent.com/yhirose/cpp-httplib/master/httplib.h"

Tor .onion address not generated

# Check Tor logs
$ sudo journalctl -u tor -n 50

# Fix permissions on hidden service dir
$ sudo chown toranon:toranon /var/lib/tor/centurion/
$ sudo chmod 700 /var/lib/tor/centurion/
$ sudo systemctl restart tor
$ sleep 5 && sudo cat /var/lib/tor/centurion/hostname
STILL STUCK? — Check logs/centurion.log for detailed backend errors, and run sudo journalctl -u centurion -f for live systemd output.
